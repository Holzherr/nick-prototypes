Open-source repo description (personal)

Repo tagline / About field

A Claude Code skill that publishes your prototypes to a live URL and lets anyone leave visual feedback right on the page — then feeds it back to Claude to iterate. Built on Plannotator.

README intro

Prototype → publish → get feedback → iterate, without leaving the loop.

This is a Claude Code skill that turns a local prototype into a live, shareable website in one step (via GitHub Pages), then drops a feedback widget into every page. Anyone with the link — teammates, clients, users — can click a button and annotate the prototype visually: point at a component, type what's wrong, done.

Copy the feedback URL back to Claude and it knows exactly which elements you're referring to, then iterates on them directly. No screenshots, no "the third card from the top," no translation between what a reviewer saw and what the model changes.

It's built for the way product builders, designers, and agencies actually work: fast reps, real feedback, tight loops. It stands on the shoulders of Plannotator for the visual-annotation layer; this skill adds instant publishing and the round-trip back into Claude.

What you get

  • One-command publish of any prototype to a live GitHub Pages URL
  • A feedback button injected into every prototype — visual, point-and-comment annotations
  • Shareable links so anyone can review without installing anything
  • Annotations that Claude can read back and act on, component by component
