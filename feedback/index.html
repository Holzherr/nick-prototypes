Memory — extracted from chat

When you chat with GitLaw, it extracts and remembers context about you. Test by having a real-ish session, then running /print-memory to see what it stored. Look for legal mistakes, misclassifications, things that shouldn't be there, things that should be there but aren't.

Important angle to test: memory should only store things that are materially valuable for future interactions. It should not store information that's already in playbooks, duplicative context, or generic chitchat. Filling memory with fluff actively degrades agent quality (it dilutes the useful signal). Please flag anything you see stored that doesn't meet a "would this uniquely help a future turn?" bar.

How to give feedback

  • Format: Each prototype has a single shared Google Doc. Find your name (or add a section under your name), write your thoughts. Other people's responses are visible — that's intentional, see what others noticed and add what's different.
  • Screenshots: Paste inline (Insert → Image or just Cmd+V). Annotate if helpful.
  • Length: Bullet points are fine. Long-form is fine. Whatever helps you express the thought clearly.
  • If you only have 15 minutes: Do P3 Memory only — that's the highest-priority one. Look at /print-memory output after one chat session and tell us what's wrong about it.
  • If something's broken: Just write it down. Don't try to fix it, don't worry about whether it's known. Raw observations are gold.
  • Stuck or want to talk it through: DM Nick in Slack. A 30-min call works if async is hard for any prototype.
Materials are public-internet visible (this site is hosted on GitHub Pages). The prototypes themselves don't contain client data. If anything sensitive needs to be said, use the Google Doc (only people with the link can read) or Slack DM.
