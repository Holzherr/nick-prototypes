Lawyer access

Two ends of the implementation-difficulty axis. BYO is the easiest to ship (no marketplace, no vetting — user supplies the lawyer). AI agent is the hardest (autonomous advice with UPL, trust and accuracy constraints).

🟢 easiest to build human-in-loop

P1 Lawyer loop-in — bring-your-own

The user invites their existing lawyer (their firm's GC, a friend who's a solicitor, etc.) to join the document. GitLaw becomes the workspace; the lawyer becomes a Guest collaborator. No marketplace, no tier system, no liability for matching.

P2 Legal-advice agent

An AI agent gives the user contract advice without routing to a human. Surfaces escalation paths and "human review needed" checkpoints — but the loop stays in chat. Scales infinitely if it works; constrained by UPL, accuracy, and trust.
